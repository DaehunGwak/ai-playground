# clap-triton vs clap-fastapi 성능 비교 리포트

- **테스트 일시**: 2026-03-08
- **테스트 도구**: k6 (동일 스크립트, BASE_URL 환경변수로 대상 전환)

---

## 1. 아키텍처 비교

| 항목 | clap-triton | clap-fastapi |
|------|-------------|--------------|
| 포트 | 8080 | 8100 |
| 구조 | FastAPI 게이트웨이 → Triton Inference Server (2-tier) | FastAPI 단일 서버 (1-tier) |
| 추론 방식 | gRPC → Triton (dynamic batching) | `laion_clap.CLAP_Module` 직접 로드 |
| 동시성 제약 | `threading.Semaphore(2)` | `asyncio.Semaphore(2)` |
| 텍스트 모델 | Triton `clap_text` (max_batch=32, delay 100ms) | CLAP_Module (in-process) |
| 오디오 모델 | Triton `clap_audio` (max_batch=8, delay 200ms) | CLAP_Module + librosa (in-process) |

## 4. 종합 분석

### 전체 성능 요약

| 항목 | 텍스트 우위 | 오디오 우위 |
|------|------------|------------|
| 처리량 | clap-fastapi (+50%) | clap-fastapi (+35%) |
| p(95) 레이턴시 | clap-fastapi (-36%) | clap-fastapi (-25%) |
| 안정성 (에러율) | 동일 (0%) | 동일 (0%) |
| Checks 통과율 | 동일 (100%) | 동일 (100%) |

### 주요 관찰

1. **clap-fastapi가 두 엔드포인트 모두에서 우위**: 텍스트는 처리량 +50%, 오디오는 +35%로 단순 단일 서버 구조가 동일 semaphore 조건에서 더 높은 처리량을 기록함.

2. **Triton 2-tier 오버헤드**: clap-triton은 FastAPI → gRPC → Triton 경로를 거치면서 네트워크 홉이 추가된다. 텍스트 p(95) 3.61s vs 2.32s 차이의 상당 부분이 이 오버헤드에서 기인한 것으로 추정.

3. **Dynamic batching의 실질 효과 부재**: Triton의 dynamic batching(텍스트: max_batch=32, delay 100ms / 오디오: max_batch=8, delay 200ms)은 이론상 배치 효율을 높여야 하지만, `semaphore=2` 제약으로 인해 Triton에 동시에 들어가는 요청이 최대 2개로 제한되어 배치 효과가 제한적이었다. 배치 지연(100~200ms)이 오히려 레이턴시를 높이는 역효과 가능성도 있음.

4. **오디오 librosa 처리**: clap-fastapi의 librosa 오디오 전처리가 in-process로 실행되어도 clap-triton보다 빠른 이유는, Triton 구조에서 오디오 전처리 자체가 Python 코드로 실행되고 그 결과를 gRPC로 전송하는 직렬 오버헤드가 더 크기 때문으로 판단됨.

5. **네트워크 비교**: 오디오 테스트에서 clap-fastapi가 629 MB 송신(vs clap-triton 466 MB)으로 더 많은 데이터를 처리했음에도 더 빠른 레이턴시를 기록, 대역폭보다 컴퓨팅 경로가 병목임을 확인.

6. **동일 안정성**: 두 서비스 모두 에러율 0%, Checks 100% 통과로 기능적 안정성은 동등. 운영 관점에서 clap-fastapi가 구성 단순성(단일 컨테이너) 측면에서 추가 이점.

### 아키텍처 선택 가이드

| 상황 | 권장 아키텍처 |
|------|--------------|
| 낮은 레이턴시 / 높은 처리량 우선 | **clap-fastapi** |
| 모델 관리 / A/B 테스트 / 버전 관리 필요 | clap-triton (Triton 모델 저장소 기능 활용) |
| 대규모 배치 추론 (batch_size > 2) | clap-triton (semaphore 제약 제거 시 dynamic batching 효과) |
| 운영 단순성 / 빠른 배포 | **clap-fastapi** |
| GPU 클러스터 스케일아웃 | clap-triton (Triton 멀티 인스턴스 지원) |
